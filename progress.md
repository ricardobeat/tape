## Root Cause Analysis: Top Failure Categories

### 1. Function `.name` not set on JS functions (~2000+ tests)
**Impact**: Every test checking `fn.name` or property descriptors on function objects.
**Status**: тЬЕ FIXED тАФ CLOSURE opcode now creates `.name` property from template name.
Anonymous functions get `""` per ES6 spec. `.length` flags also fixed to `PROP_FLAGS_NWC`
(non-writable, non-enumerable, configurable) per ES5 ┬з15.3.5.1.

### 2. Lightfunc constructors lack `.prototype` (~1500+ tests, cascading)
**Root cause**: Built-in constructors (Boolean, Number, String, Array, Object,
Function, Error, etc.) are lightfuncs. The GETPROP lightfunc handler (`vm.c3:2213`)
resolves `.name` and `.length`, but for "prototype" it walks `Function.prototype`
which doesn't have `.prototype` тЖТ returns `undefined`.
**Cascade**: `Boolean.prototype` === `undefined` тЖТ `Object.getOwnPropertyDescriptor(
Boolean.prototype, 'toString')` тЖТ "called on non-object" тАФ a single root cause
explains BOTH the missing `.prototype` and the getOwnPropertyDescriptor errors.
**Status**: тЬЕ FIXED (Session 101) тАФ Added `.prototype` case in lightfunc GETPROP
handler with `lightfunc_get_proto()` helper mapping fn_index тЖТ heap proto fields.
Also fixed INSTANCEOF register aliasing bug (ra/rb sharing same register).

### 3. Built-in method `.name` absent on lightfunc methods (~800+ tests)
**Impact**: `Boolean.prototype.toString.name` should be "toString". Lightfunc
methods don't have `.name` as an own property (GETPROP resolves it for lightfuncs
but it's not in the object's property table тАФ so `getOwnPropertyDescriptor` returns null).
**Status**: тЬЕ FIXED (Session 101) тАФ `getOwnPropertyDescriptor` handler synthesizes
`.name` and `.length` descriptors for lightfuncs (builtins.c3:4303-4384).

### 4. Property descriptor flags wrong on built-ins (~600+ tests)
**Impact**: Built-in methods were registered with `PROP_FLAGS_WC` (writable:true,
configurable:true) but spec requires `writable:false, configurable:true`.
`.length` on built-ins was `PROP_FLAGS_WEC` but spec requires `writable:false`.
**Status**: тЬЕ FIXED (Session 102) тАФ Changed all method registrations to `PROP_FLAGS_NWC`
(non-writable, non-enumerable, configurable). Kept `.constructor` as `PROP_FLAGS_WC`
(writable:true per spec). Kept `Error.prototype.name/message` as `PROP_FLAGS_WC`.
Changed `.length` on RegExp object and function objects to `PROP_FLAGS_NWC`.
