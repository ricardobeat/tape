## Root Cause Analysis: Top Failure Categories

### 1. String `===` comparison using pointer identity (~fixed, but cascading effects)
**Impact**: Every test using `===` on non-interned strings (e.g., `str[0] === "h"`).
**Root cause**: The `SEQ` opcode compares strings by `get_heapptr() == get_heapptr()`
(pointer identity), which is correct only if all strings are properly interned. The
GETPROP handler for string primitive index access (`str[i]`) and string object index
access allocated new HString objects without looking up the string table first, producing
different pointers than the interned copies of the same content.
**Status**: ✅ FIXED (Session 103) — Added `Heap.str_intern()` method that does
lookup-first. Replaced all `hstring_alloc` + `str_table_insert` call sites with
`str_intern()` in vm.c3, builtins.c3, and heap.c3. Also simplified `intern_string()`,
`vm_number_to_string()`, and `vm_fastint_to_string()` to use the new helper.

### 2. Function `.name` not set on JS functions (~2000+ tests)
**Impact**: Every test checking `fn.name` or property descriptors on function objects.
**Status**: ✅ FIXED — CLOSURE opcode now creates `.name` property from template name.

### 3. Lightfunc constructors lack `.prototype` (~1500+ tests, cascading)
**Status**: ✅ FIXED (Session 101) — Added `.prototype` case in lightfunc GETPROP
handler with `lightfunc_get_proto()` helper mapping fn_index → heap proto fields.

### 4. Built-in method `.name` absent on lightfunc methods (~800+ tests)
**Status**: ✅ FIXED (Session 101) — `getOwnPropertyDescriptor` handler synthesizes
`.name` and `.length` descriptors for lightfuncs.

### 5. Property descriptor flags wrong on built-ins (~600+ tests)
**Status**: ✅ FIXED (Session 102) — Changed all method registrations to `PROP_FLAGS_NWC`.

### 6. String wrapper indexed properties missing (~70+ tests)
**Impact**: `Object.keys(new String("abc"))`, `for-in`, `getOwnPropertyDescriptor`,
`hasOwnProperty` all returned empty results for String wrapper objects.
**Root cause**: The String constructor only set `.length` as an own property.
ES5 §15.5.5.1 requires indexed character properties ("0", "1", ...) as own
non-writable, enumerable, non-configurable properties.
**Status**: ✅ FIXED (Session 104) — Added `populate_string_indexed_props()` helper
that eagerly creates indexed properties. Applied to all 5 String wrapper creation
sites: `builtin_string()`, `builtin_to_object()`, `builtin_object()`, INITFOR,
and with-statement. Also restructured `getOwnPropertyDescriptor` to run the
string exotic fallback AFTER the default lookup (per spec). Added `PROP_FLAGS_NEC`
constant for non-writable/enumerable/non-configurable.

## Session History (condensed)

### Session 112: Map/Set iterator protocol (ES6 §23.1.3, §23.1.5, §23.2.3, §23.2.5)
Implemented full ES6 Map/Set iterator protocol:
- **Object classes**: Added `MAP_ITERATOR` and `SET_ITERATOR` to ObjClass enum with shared
  `iterator_target`/`iterator_index`/`iterator_kind` fields (reuses existing HObject fields).
- **BUILTIN constants**: Added `BUILTIN_MAP_ITERATOR_NEXT=287`, `BUILTIN_SET_ITERATOR_NEXT=288`,
  `BUILTIN_MAP_PROTO_SYMBOL_ITER=289`, `BUILTIN_SET_PROTO_SYMBOL_ITER=290`.
- **`create_map_iterator(ctx, kind)`**: Creates a `MAP_ITERATOR` object with `prototype = %IteratorPrototype%`,
  stores a `.next` own property pointing to `BUILTIN_MAP_ITERATOR_NEXT`. Kind: 0=values, 1=keys, 2=entries.
- **`create_set_iterator(ctx, kind)`**: Same pattern for `SET_ITERATOR` with `BUILTIN_SET_ITERATOR_NEXT`.
- **`builtin_map_iterator_next()`**: Steps through Map entries array `[k0,v0,k1,v1,...]`, returns
  `{value, done}` objects per spec. Entries format creates `[key, value]` arrays with proper `.length`.
- **`builtin_set_iterator_next()`**: Steps through Set values array `[v0,v1,...]`. Entries format creates
  `[value, value]` pairs.
- **Rewrote Map.prototype methods**: `keys()`, `values()`, `entries()` now return proper `MAP_ITERATOR` objects.
- **Rewrote Set.prototype methods**: `keys()`, `values()`, `entries()` now return proper `SET_ITERATOR` objects.
- **`Map.prototype[@@iterator]`**: Registered as `entries()` via `BUILTIN_MAP_PROTO_SYMBOL_ITER`.
- **`Set.prototype[@@iterator]`**: Registered as `values()` via `BUILTIN_SET_PROTO_SYMBOL_ITER`.
- **GC marking**: Added cases for `ARRAY_ITERATOR`/`MAP_ITERATOR`/`SET_ITERATOR` (mark `iterator_target`)
  and `STRING_ITERATOR` (mark `extra.prim.primitive_value`) in `drain_gray()`.
- **ICEntry fix**: Added `prop_value_ptr` field to ICEntry struct for inline cache direct pointer access.
- Verified: `map.keys().next()` → `{value, done}`, `iter[Symbol.iterator]() === iter` via %IteratorPrototype%.
  No regressions in quick.sh (177/106/56).
Fixed `get_prop_key()` in vm.c3 to handle OBJECT tag by unwrapping String/Number/Boolean
wrapper objects via `to_primitive_value_local()` (made `@public` in builtins.c3). Before
this fix, `(new String("foo")) in { "foo": 1 }` returned `false` because the wrapper was
not unwrapped — `get_prop_key` fell through to the default case returning "undefined" as
the property key. Added OBJECT case that unwraps the value and recurses for correct
ToPropertyKey conversion. +8 Phase 5, +5 Phase 7, +15 Phase 8 test262 passes. Also
discovered pre-existing Number wrapper valueOf() bug (returns NaN) — added to deferred.

### Session 110: Array.from (ES6 §22.1.2.1)

Implemented `Array.from(items[, mapfn[, thisArg]])` with dual-path algorithm:
- **Iterable path**: Uses `@@iterator` (Symbol.iterator) via `get_prop_proto` + `heap.call_fn` to iterate
  any iterable (arrays, strings, etc.). Calls `.next()` in a loop, checks `.done`/`.value`, applies
  mapfn if provided.
- **Array-like path**: For non-iterable objects — validates items is not null/undefined, reads `.length`,
  iterates indexed properties via `get_prop_proto`. Strings handled specially with direct char access.
- **Error handling**: TypeError for non-callable mapfn, TypeError for null/undefined items.
- **Metadata**: Registered `BUILTIN_ARRAY_FROM=285` with arity=1, registered as static method on
  `Array` constructor via lightfunc. +12 test262 passes in Phase 5 (2410 from 2398).
- Verified both nanbox and nonanbox builds pass. No regressions in `quick.sh` (177/106/56).

### Session 111: %IteratorPrototype% shared prototype (ES6 §25.1.2)

Implemented `%IteratorPrototype%` — the shared prototype for all built-in iterators:
- **Heap field**: Added `iterator_proto` to Heap struct with init and GC mark.
- **`%IteratorPrototype%` object**: Created in `register_iterator_prototype()` before prototype wiring,
  stored in `heap.iterator_proto`. Prototype set to `Object.prototype` during wiring.
- **`[Symbol.iterator]()` method**: Added `BUILTIN_ITERATOR_PROTO_SYMBOL_ITER` (builtin index 286)
  that returns `this` — enables `iter[Symbol.iterator]() === iter` per spec.
- **Wired to iterators**: Changed `iter.prototype = null` to `iter.prototype = heap.iterator_proto`
  for both `ARRAY_ITERATOR` and `STRING_ITERATOR` creation sites (3 locations: `builtin_array_proto_symbol_iter`,
  `create_array_iterator`, `builtin_string_proto_symbol_iter`).
- **Dependency for Map/Set iterators**: This prototype is required before Map/Set iterator methods
  can return proper iterator objects instead of Arrays.
- Verified: `Object.getPrototypeOf(iter) !== null` now true, `iter[Symbol.iterator]() === iter` works.
  No regressions in quick.sh (177/106/56).

### Session 109: Fix Number wrapper valueOf/toPrimitive fastint corruption
Fixed 3 identical bugs where `primitive_value.get_number()` was called when the TVal might
hold a fastint tag. `get_number()` blindly bitcasts nanboxed bits as a double, producing
garbage for fastint values (e.g., `new Number(42).valueOf()` returned NaN). Changed all 3
call sites to use `get_number_or_fastint()`: (1) `vm.c3:630` to_primitive_value(),
(2) `builtins.c3:596` to_primitive_value_local(), (3) `builtins.c3:3790`
builtin_number_proto_valueOf(). Also optimized valueOf to preserve fastint tag when possible
for small integers. Verified both nanbox and non-nanbox builds pass. +14 Phase 3, +2 Phase 8
test262 passes.

### Session 107: RegExp.prototype accessor properties (ES6 §21.2.5)
Implemented 8 accessor getter properties on RegExp.prototype: `flags`, `source`,
`global`, `ignoreCase`, `multiline`, `dotAll`, `sticky`, `unicode`. These are
now getter accessor properties (non-enumerable, configurable) instead of own
data properties on each RegExp instance. (1) Added `register_proto_accessor()`
helper that creates a builtin function object with `.name = "get <prop>"` and
`.length = 0`, then registers it as an accessor on the prototype. (2) Modified
`invoke_getter()` in vm.c3 to handle builtin function getters directly without
creating a JS call frame — fast path for C3 builtin getters. (3) Extended
js_flags encoding: 1=g, 2=i, 4=m, 8=s, 16=u, 32=y. (4) Removed own-property
creation for source/global/ignoreCase/multiline from both `builtin_regexp()` and
VM NEWREGEXP opcode. `lastIndex` remains an own data property. (5) Updated
`builtin_regexp_proto_toString()` to output s/u/y flags. (6) Added
`regexp_get_this_or_throw()` helper for TypeError on non-RegExp receivers.

### Session 106: JSON.stringify replacer/toJSON/edge cases (ES5 §15.12.3)
